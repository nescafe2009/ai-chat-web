# 星辰 (Stellaris) - 规章/价值观在任务前生效的护栏方案

日期: 2026-02-17
状态: 已批准

## 0. 目标与约束

**目标:** 保证组织愿景、核心价值观、执行规章在每次任务开始前都被显式考虑，并能对高风险动作形成可审计的约束。

**约束:**
- 不依赖"记忆/习惯"，依赖流程与技术注入
- 不显著拖慢日常低风险迭代
- 对高风险变更提供"可阻断/可审计/可回滚"的机制

## 1. 护栏分层（从强到弱）

本方案采用"四层护栏"叠加，上层失败时由下层兜底。

### 1.1 系统级护栏（Always-on 规则注入）

**核心思想:** 让 Agent 每次推理都能看到一份"规则精简版"，作为默认上下文的一部分。

**实现方式:**
- 维护精简规则文本：`docs/stellaris-rules-lite.md`
- 在 AGENTS.md 中引用，每次会话自动加载
- 内容范围：愿景/价值观/执行规章/例外条款

### 1.2 任务级护栏（Preflight 检查模板）

**核心思想:** 每次任务开始前，先输出一个短的 preflight。

**日常精简版（3 项）:**
1. 目标
2. 风险等级：low / medium / high
3. 是否触发例外条款

**完整版（Medium/High 时展开）:**
4. 回滚方案
5. 验收标准
6. 影响范围

### 1.3 工具级护栏（对高风险动作的强制确认/阻断）

**例外条款清单（触发时需老板确认）:**
- 权限/认证/ACL
- 公网暴露（端口、反向代理、token/secret）
- 不可逆数据变更（删除、迁移、格式变更）
- 安全边界变化（firewall、trusted proxies、CORS、webhooks）
- 模型/API/embedding 配置变更
- cron/定时任务变更
- 第三方集成/授权

**落地方式:** PR 模板为主，脚本门禁为辅

### 1.4 产物级护栏（档案/PR 审计）

- 组织正式文件必须进入 `docs/` 并走 PR/Review 定稿
- 文档必须包含 YAML frontmatter 元信息
- 定稿后修改必须通过 PR

## 2. 验收标准

- 任意一次任务开始，都能看到 preflight 输出
- 高风险动作在未确认前不会被直接执行
- 组织正式文件可在 `docs/` 中定位，且变更有 PR/commit 记录
